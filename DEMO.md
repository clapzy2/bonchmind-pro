# BonchMind Pro — демо-сценарий

Пошаговый сценарий показа продукта (для защиты / портфолио). Рассчитан на ~5 минут. На каждом шаге — что сделать, что показать и слот под скриншот.

> Перед показом подними проект — локально (`python run_api.py` + `npm run dev`) или через Docker (`docker compose up`). Подробности в [`README.md`](README.md). Первый конспект/ответ может быть медленнее обычного (прогрев моделей).

---

## 0. Пара слов о проекте (10 секунд)

> «BonchMind Pro — это RAG-платформа по учебным материалам. Каждый пользователь загружает свои документы в личное изолированное пространство, генерирует по ним конспекты и задаёт вопросы ассистенту, который отвечает с опорой на источники, а не выдумывает.»

---

## 1. Регистрация → личное пространство

**Действие:** открыть `http://localhost:3000`, зарегистрироваться (например `alice@example.com`).

**Что показать:** после регистрации пользователь сразу попадает в рабочую область — backend автоматически создал персональный workspace. Токен живёт в HttpOnly-cookie, в `localStorage` ничего не хранится.

![Экран входа / регистрации](screenshots/01-login.png)

---

## 2. Загрузка материала

**Действие:** вкладка **Библиотека** → загрузить учебный файл (`.pdf` / `.txt` / `.docx`).

**Что показать:** прогресс индексации в реальном времени; по завершении материал получает статус **«Готов»**. Под капотом: файл сохранён, создана запись `Document`, текст разбит на чанки, проиндексирован в ChromaDB с `workspace_id` в metadata.

![Библиотека с загруженным материалом](screenshots/02-materials.png)

---

## 3. Конспект по теме

**Действие:** вкладка **Конспект** → выбрать материал, ввести тему (например `Wi-Fi`) → **Сгенерировать**.

**Что показать:**
- конспект собирается **только по найденным фрагментам** выбранного материала;
- кнопка **«Проверить источники»** показывает, какие разделы/фрагменты поддержали каждый пункт;
- **«Экспорт»** выгружает конспект в `.docx`.

![Сгенерированный конспект](screenshots/03-summary.png)

---

## 4. Ассистент по материалам

**Действие:** вкладка **Ассистент** → задать вопрос по загруженному материалу (например «Что такое Wi-Fi?»).

**Что показать:** ответ привязан к содержимому материала (а не к общим знаниям модели). Можно вести тему короткими follow-up вопросами — ассистент держит контекст диалога.

![Ассистент: вопрос и ответ](screenshots/04-assistant.png)

---

## 5. Изоляция пользователей (ключевое)

**Действие:** выйти, зарегистрировать второй аккаунт (`bob@example.com`), зайти.

**Что показать:** у Bob **пустая библиотека** — материалы Alice ему не видны. Это и есть multi-user изоляция: `workspace_id` берётся из авторизации, а не из запроса, поэтому пользователь физически не может обратиться к чужому пространству.

![Изоляция: пустое пространство второго пользователя](screenshots/05-isolation.png)

---

## 6. (Опционально) Production-запуск в Docker

**Действие:** показать `docker compose up` — Postgres + backend + frontend в контейнерах.

**Что показать:** тот же продукт работает на PostgreSQL вместо SQLite, миграции применяются автоматически при старте, данные переживают перезапуск (тома). Подробно — раздел «Запуск через Docker» в [`README.md`](README.md).

![Стек в Docker](screenshots/06-docker.png)

---

## Что подчеркнуть на защите

- **RAG с опорой на источники** — ответы/конспекты строятся из реальных фрагментов, видно покрытие.
- **Multi-user изоляция** — центральный инвариант, `workspace_id` только из auth.
- **Полный стек** — Next.js + FastAPI + Postgres + ChromaDB + BGE-модели, упаковано в Docker.
- **Инженерная дисциплина** — тесты, CI, миграции, поэтапная разработка через PR.

Архитектура подробно — в [`ARCHITECTURE.md`](ARCHITECTURE.md).
