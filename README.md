# BonchMind Pro

BonchMind Pro - локальная RAG-система для работы с учебными материалами.

Проект уже умеет:
- загружать и индексировать учебные файлы;
- строить конспекты по теме или по материалу;
- отвечать на вопросы по базе;
- показывать, на каких фрагментах держится ответ;
- экспортировать конспекты в `.docx`;
- работать как multi-user приложение с auth + изолированными workspace через `Next.js + FastAPI`.

Сейчас это не просто "демо с чатиком", а уже рабочая учебная станция, которую мы постепенно превращаем в сильный продукт.

---

## Как запустить

Основной интерфейс — Next.js на FastAPI с аутентификацией (Stage 5).

Запуск backend:

```powershell
python run_api.py
```

Запуск frontend:

```powershell
cd frontend
npm install
npm run dev
```

Открыть:

```text
http://127.0.0.1:3000
```

Дальше — `/register` или `/login`. Подробнее про auth и поведение UI — в [frontend/README.md](frontend/README.md).

### Текущие вкладки нового UI

- `Конспект` — генерация конспекта, выбор материала, проверка покрытия, экспорт DOCX;
- `Ассистент` — диалог по материалам с режимами ответа;
- `Материалы` — библиотека источников, загрузка, удаление, переиндексация;
- `Проверка качества` — разбор силы ответа, покрытия и слабых мест;
- `Настройки` — пока заготовка под системные и продуктовые настройки.

### Legacy Gradio UI

Удалён в Stage 6d (`main.py`/`run.py` + `gradio` dependency). Причина — он обходил Stage 3+ auth и писал всё в `config.DEFAULT_WORKSPACE_ID` минуя `Document` table, поэтому workspace-изоляция, верифицированная `tests/test_two_users_isolation.py`, через него не работала. Если потребуется быстрый maintenance reset базы — отдельный CLI-скрипт, а не UI-кнопка.

## Что еще в работе

Это важный раздел. Да, в `README` будет и должно быть написано, что сделано и что еще проблемное.

### Продукт еще не завершен

Сейчас проект находится между стадиями:
- "уже реально работает",
- "еще не доведен до финального polished-продукта".

### Что еще сыровато

- часть экранов еще перегружена служебными блоками и нуждается в дальнейшей полировке;
- `Настройки` пока не полноценный пользовательский раздел;
- `Проверка качества` скорее инструмент владельца/разработчика, чем конечного пользователя;
- ассистент уже рабочий, но UX еще можно усиливать;
- нет многопользовательской архитектуры, ролей и авторизации;
- нет нормального production-ready хранения пользовательских сессий;
- не все ошибки backend/frontend еще объясняются пользователю идеально;
- долгие операции индексирования и генерации еще требуют более красивого прогресса и контроля.

### Известные ограничения

- первый запуск может быть заметно долгим из-за прогрева моделей;
- локальный режим через Ollama сильно зависит от мощности ПК;
- качество конспекта напрямую зависит от качества исходного текста и структуры документа;
- у маленьких текстовых файлов может не быть нормальных "разделов" - это не всегда ошибка, иногда это естественное поведение;
- frontend в dev-режиме требует установленный Node.js и `npm` в PATH.

## Режимы LLM

BonchMind Pro поддерживает два режима работы языковой модели.

### Локальный режим через Ollama

Пример `.env`:

```env
LLM_MODE=ollama
OLLAMA_MODEL=qwen2.5:14b
```

Перед запуском нужно установить Ollama и скачать модель:

```powershell
ollama pull qwen2.5:14b
```

Плюсы:
- данные остаются локально;
- можно работать без внешнего API;
- удобно для локальной разработки.

Минусы:
- нужен более мощный компьютер;
- ответы могут генерироваться дольше.

### API-режим

Пример `.env`:

```env
LLM_MODE=api
API_KEY=your_api_key_here
API_MODEL=qwen/qwen3-32b
```

Плюсы:
- выше скорость старта;
- нет нагрузки на локальную машину;
- можно получить более сильную модель.

Минусы:
- нужен интернет;
- нужен API-ключ;
- данные уходят во внешний сервис.

## Установка

### 1. Клонирование

```powershell
git clone https://github.com/clapzy2/bonchmind-pro.git
cd bonchmind-pro
```

### 2. Python-окружение

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

### 3. `.env`

Создайте `.env` на основе `.env.example`.

Минимальный локальный пример:

```env
LLM_MODE=ollama
OLLAMA_MODEL=qwen2.5:14b
```

### 4. Frontend-зависимости

Только для нового интерфейса:

```powershell
cd frontend
npm install
```
