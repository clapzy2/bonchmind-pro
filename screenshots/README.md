# Скриншоты

Сюда кладутся скриншоты интерфейса, на которые ссылаются [`README.md`](../README.md) и [`DEMO.md`](../DEMO.md). PNG снимаются с живого интерфейса вручную.

## Shot-list

Сними и положи сюда файлы с **точно такими именами** (тогда ссылки в доках заработают):

| Файл | Что на скриншоте | Где снять |
|------|------------------|-----------|
| `01-login.png` | Экран входа / регистрации | `/login` или `/register` |
| `02-materials.png` | Библиотека с загруженным материалом (статус «Готов») | вкладка **Библиотека** |
| `03-summary.png` | Сгенерированный конспект по теме | вкладка **Конспект** после генерации |
| `04-assistant.png` | Ассистент: вопрос и ответ с диалогом | вкладка **Ассистент** после ответа |
| `05-isolation.png` | Пустая библиотека второго пользователя | войти вторым аккаунтом |
| `06-docker.png` | (опц.) `docker compose up` — стек в контейнерах | терминал / Docker Desktop |

## Как снимать

- Полноэкранный браузер, тёмная тема (она единственная), окно пошире — чтобы был виден трёхколоночный layout.
- Перед `03`/`04` сгенерируй реальный конспект/ответ, чтобы на экране был контент, а не пустые состояния.
- Формат — PNG. Можно дополнительно положить версии под README (например ужать до ~1600px по ширине).

После добавления файлов скриншоты автоматически отрендерятся в README и DEMO на GitHub.
